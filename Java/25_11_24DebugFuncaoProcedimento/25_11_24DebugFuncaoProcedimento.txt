PARTE 2: DÚVIDAS ESPECÍFICAS (Respostas Consolidadas)
--------------------------------------------------------------------------------

P: O nome da variável parametroNome poderia ser simplesmente nome?
R: Sim. O escopo das variáveis é local. A variável 'nome' do main e o parâmetro 'nome' da função são espaços de memória diferentes.

P: O que acontece se chamarmos exibirSaudacao() sem parâmetros?
R: Erro de Compilação. O Java exige que a assinatura do método seja respeitada. Se o método pede uma String, é obrigatório enviar uma String.

PARTE 3: DESAFIO PRÁTICO (CORREÇÃO DE BUG)
--------------------------------------------------------------------------------
Problema: O método era 'void' mas tentava-se atribuir seu valor a uma variável.
Sua Solução (Correta): Alterar para tipo com retorno e adicionar 'return'.

public class ConversorTemperatura {
    public static void main(String[] args) {
        double temperaturaCelsius = 30.0;
        
        // Agora funciona pois o método retorna um valor
        double temperaturaFahrenheit = converterParaFahrenheit(temperaturaCelsius); 
        
        System.out.println("A temperatura final é: " + temperaturaFahrenheit);
    }

    // Método corrigido de 'void' para 'Double' (ou double) e com return
    public static Double converterParaFahrenheit(double celsius) {
        double resultado = (celsius * 1.8) + 32;
        System.out.println("Convertendo...");
        return resultado;
    }
}
